## 三、关键技术实现说明

### 3.1 OCR / 图片分析能力

赛题要求支持"软件界面操作截图"等多格式成果。本系统采用**多模态大模型替代传统 OCR** 的技术路线：

| 方案对比 | 传统 OCR（Tesseract） | 本系统方案（多模态大模型） |
|---------|---------------------|------------------------|
| 文字识别 | 仅识别印刷体文字 | 识别文字 + 理解图表/截图/UI界面 |
| 上下文理解 | 无 | 结合实训任务上下文理解图片内容 |
| 部署依赖 | 需安装 Tesseract 引擎 + 语言包 | 无需额外依赖，通过 API 调用 |
| 兜底策略 | 本地离线可用 | 当前依赖云端 API，未来可扩展 Tesseract 本地兜底 |

**实现流程：**
1. 图片上传后，`DocumentTextExtractor` 识别为图片类型，返回占位标记
2. `AiService.doParse` 检测到图片文件，调用 `ModelScopeClient.analyzeImage`
3. Spring AI 的 `Media` + `UserMessage` 将图片以 base64/文件流方式发送给视觉模型
4. 使用固定 Prompt：`"请分析这张学生提交图片中的文字、图表、代码或实验结果，提取可用于核查和评分的关键信息。"`
5. 分析结果与文本解析结果合并，进入核查和评分环节

**配置文件：** `application.yml` 第 71 行 `ai.vision-model: Qwen/Qwen3.5-35B-A3B`

### 3.2 RAG 知识库检索增强

| 环节 | 实现 |
|-----|------|
| 文档入库 | 上传 → 文本提取 → 1200 字符分块 → 向量化（damo/nlp_corom_sentence-embedding_chinese-base） |
| 检索策略 | 两阶段：向量相似度 Top-20 → 可选 Rerank 精排 Top-5 |
| 应用场景 | AI 评分时注入相关知识片段，提升评分准确性和一致性 |

### 3.3 异步任务调度

| 特性 | 实现 |
|-----|------|
| 调度方式 | `@Scheduled(fixedDelay=5000)` 数据库轮询（非消息队列，降低部署复杂度） |
| 任务类型 | PARSE（解析）、CHECK（核查）、SCORE（评分）、EXPORT（导出） |
| 容错机制 | 最多重试 3 次，递增延迟（30s × 重试次数） |
| 僵尸清理 | 运行超过 10 分钟的任务自动标记为失败 |
| 进度追踪 | `BatchProgress.vue` 页面实时显示异步任务进度 |

### 3.4 权限与安全

| 层级 | 实现 |
|-----|------|
| URL 级 | Spring Security 配置：`/api/auth/**` 放行，其余需 JWT |
| 方法级 | `@PreAuthorize("hasRole('ADMIN/TEACHER/STUDENT')")` |
| 数据级 | Service 层按角色过滤（学生只看自己、教师只看自己课程） |
| 文件级 | `FileController.hasFileAccess` 通过 submissionId 关联验证 |
| 前端 | Vue Router 守卫 + localStorage Token + 401/403 自动跳转登录 |
| 审计 | `operation_log` 表记录用户操作（IP、路径、方法、时间） |

## 四、部署与国产化适配

### 4.1 当前运行环境

| 组件 | 版本/配置 |
|-----|----------|
| 操作系统 | Windows 11（开发环境） |
| JDK | 17（Spring Boot 3.4.3 要求） |
| 数据库 | MySQL 8.0（utf8mb4） |
| AI 服务 | ModelScope 云端 API |

### 4.2 国产化适配方案

赛题要求部署在 **LoongArch 架构 CPU + 银河麒麟高级服务器操作系统 V10/V11** 上。适配方案如下：

**后端适配：**
- Spring Boot 3 基于纯 Java 17 编写，无平台相关 native 代码，LoongArch 平台 JDK 17 可直接运行
- `mvn clean package -DskipTests` 生成的 fat jar 为平台无关产物，`java -jar` 即可启动
- 所有依赖（MyBatis-Plus、Spring AI、PDFBox、POI、iText 等）均为纯 Java 库，无需 native 编译

**前端适配：**
- Vite 构建产物为纯静态 HTML/CSS/JS，无平台依赖
- 部署到 Nginx（麒麟系统支持 loongarch64 版本）即可

**数据库适配：**
- 当前使用 MySQL 8.0（开发环境）
- 龙芯麒麟环境可选：
  - **MySQL 龙芯版**：龙芯社区提供 loongarch64 编译版本
  - **openGauss / PostgreSQL**：已有 `schema-migration.sql` 脚本支持迁移（需将 MySQL 语法转换为 PostgreSQL 语法，主要差异为自增 ID 和日期函数）
  - **pgvector**：向量字段当前存储为 JSON longtext，迁移到 pgvector 后可改用向量索引提升检索性能

**AI 服务适配：**
- 云端 API 模式：无需本地 GPU，任何网络可达的环境均可使用
- 本地部署模式（赛题选项）：可在 LoongArch 平台部署 Ollama 或 vLLM 推理框架，加载 Qwen 开源模型，仅需修改 `application.yml` 中的 `ai.base-url` 即可切换

### 4.3 部署步骤（麒麟 + LoongArch）

```bash
# 1. 安装 JDK 17（龙芯版）
yum install java-17-openjdk-loongarch64

# 2. 安装 MySQL / openGauss
# （根据实际环境选择）

# 3. 初始化数据库
mysql -u root -p < schema.sql

# 4. 部署后端
java -jar backend-1.0.0.jar --server.port=8080

# 5. 部署前端（Nginx）
cp -r dist/* /usr/share/nginx/html/
# 配置 nginx.conf 反向代理 /api → localhost:8080
```
