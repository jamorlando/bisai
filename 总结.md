# 实训成果智能核查与评价系统 — 项目总结

---

## 一、项目概述

### 1.1 项目名称

**实训成果智能核查与评价系统**

### 1.2 项目目标

构建一个基于 AI 的实训成果自动化核查与评分平台，实现从任务创建、学生提交、AI 智能解析/核查/评分到教师复核、成绩发布的全流程闭环管理。

### 1.3 核心价值

| 能力 | 说明 |
|------|------|
| **智能文档解析** | 支持 PDF/Word/Excel 等多种格式的自动解析与内容提取 |
| **AI 规范核查** | 从完整性、规范性、原创性、技术准确性、任务匹配度五个维度智能核查 |
| **AI 智能评分** | 结合知识库 RAG 检索结果，按评价指标自动生成评分与评语 |
| **教师复核校准** | 教师可逐项调整 AI 评分，支持评分校准与修正记录 |
| **可视化报表** | ECharts 图表展示班级成绩分布、指标达成度等统计数据 |
| **动态模型配置** | 管理后台可实时切换 AI 模型，无需重启服务 |

## 四、核心业务流程

### 4.1 主流程

```
教师创建任务 → 学生上传成果 → AI 文档解析 → AI 智能核查 → AI 智能评分 → 教师复核 → 成绩发布
   教师           学生           异步            异步           异步          教师        系统
```

1. **教师创建任务** — 设定任务要求、评价模板、截止时间、允许文件类型
2. **学生上传成果** — 支持 doc/docx/pdf/xls/xlsx/jpg/png/zip 等格式，支持重新提交
3. **AI 文档解析** — 异步提取文档内容（POI/PDFBox 文本提取 + AI 增强解析）
4. **AI 智能核查** — 基于任务要求，从完整性、规范性、原创性、技术准确性、任务匹配度核查
5. **AI 智能评分** — 结合知识库 RAG 检索结果，按评价指标逐项生成评分与评语
6. **教师复核** — 查看 AI 评分详情，可逐项校准调整，添加总评
7. **成绩发布** — 确认后学生可查看评价结果，支持导出 PDF/Word 报告

### 4.2 核心状态机

#### 提交 (Submission) — 三条并行状态流

```
parseStatus:   PENDING → PARSING → SUCCESS / FAILED
checkStatus:   NOT_CHECKED → CHECKING → SUCCESS / CHECK_FAILED
scoreStatus:   NOT_SCORED → SCORING → AI_SCORED → TEACHER_CONFIRMED → PUBLISHED
                                          ↘ SCORE_FAILED
                                          ↘ RETURNED
```

#### 训练任务 (TrainingTask)

```
DRAFT → PUBLISHED → CLOSED / ARCHIVED
```

#### 异步任务 (AsyncTask)

```
PENDING → RUNNING → SUCCESS / FAILED
                     ↗ RETRYING → RUNNING（最多 3 次重试）
```

### 4.3 异步任务调度

- `AsyncTaskService` 使用 `@Scheduled(fixedDelay=5000)` 数据库轮询
- 支持 PARSE / CHECK / SCORE / EXPORT 四种任务类型
- 最大重试 3 次（递增延迟），僵尸任务 10 分钟自动清理
- 任务带进度百分比和当前步骤描述，前端可实时展示

## 六、角色与权限体系

### 6.1 三角色能力

| 角色 | 能力范围 |
|------|----------|
| **学生 (STUDENT)** | 查看任务 · 上传成果 · 查看提交记录 · 查看评价结果 · 消息中心 |
| **教师 (TEACHER)** | 任务管理 · 提交管理 · 文件预览 · 解析/核查/评分复核 · 评价模板 · 知识库管理 · 报表中心 · 批量操作 |
| **管理员 (ADMIN)** | 用户管理 · 班级课程管理 · 知识库管理 · AI 模型配置 · 系统日志 · 评分校准 · 评价模板管理 |

### 6.2 六层权限控制

```
┌─ 1. URL 级      → Spring Security 配置：/api/auth/** 放行，其余需认证
├─ 2. 方法级      → @PreAuthorize("hasRole('ADMIN')") 等注解
├─ 3. 数据级      → Service 层按 role 过滤（学生只看自己，教师只看自己课程）
├─ 4. 文件级      → FileController.hasFileAccess 通过 submissionId 关联验证
├─ 5. 路由级      → Vue Router meta.roles 控制前端页面访问
└─ 6. 签名校验    → localStorage 防篡改机制
```

### 6.3 关键权限规则

- `CourseService.listCourses`：教师只看到自己的课程，管理员看到所有
- `TaskService.createTask`：通过 `isTeacherOwnerOfCourse` 验证教师只能在自己课程下创建任务
- `PermissionService.isStudentOwnerOfSubmission`：验证学生只能访问自己的提交
- 文件访问：通过 submissionId 关联验证权限，前端用 axios blob 下载

## 七、关键模块详解

### 7.1 AI 服务层

**核心类**: `AiService` · `ModelScopeClient`

| 方法 | 功能 |
|------|------|
| 文档解析 | 提取文档文本 → AI 增强解析 → 生成摘要/主题/建议 |
| 智能核查 | 基于任务要求，从 5 个维度（完整性、规范性、原创性、技术准确性、任务匹配度）核查 |
| 智能评分 | 结合知识库 RAG 检索结果，按评价指标逐项生成评分、理由和证据 |

**AI 调用封装**：`ModelScopeClient` 封装 ModelScope 平台 API 调用，支持任意模型切换（Qwen / DeepSeek / Kimi 等），调用记录写入 `ai_call_log` 表（Token 统计、成功/失败追踪）。

### 7.2 知识库 RAG

**核心类**: `KnowledgeService` · `KnowledgeRetrievalService`

```
文档上传 → 解析 → 1200 字符分块 → 向量化存储
                                    │
AI 评分时 ← Rerank Top-5 ← 向量 Top-20 检索
```

- **分块策略**：1200 字符/块，估算 Token 数
- **向量化**：使用 `damo/nlp_corom_sentence-embedding_chinese-base` 模型
- **两阶段检索**：先向量 Top-20 粗检索，再可选 Rerank Top-5 精排
- 为 AI 评分提供上下文增强，提升评分准确性

### 7.3 异步任务调度

**核心类**: `AsyncTaskService` · `TaskService`

| 特性 | 说明 |
|------|------|
| 调度方式 | `@Scheduled(fixedDelay=5000)` 数据库轮询（非消息队列） |
| 任务类型 | PARSE / CHECK / SCORE / EXPORT |
| 重试机制 | 最多 3 次，递增延迟 |
| 僵尸清理 | 运行超 10 分钟的任务自动标记失败 |
| 并发控制 | `TaskService` 管理批量操作并发 |
| 进度追踪 | 每个任务带 progress（0-100）和 currentStep 描述 |

### 7.4 报表生成

**核心类**: `ReportService`

- 支持生成 PDF 和 Word 格式的评价报告
- 使用 JFreeChart 1.5 生成嵌入报告的可视化图表
- 文件路径使用 `toAbsolutePath().normalize()` 存储绝对路径

### 7.5 系统配置管理

**核心类**: `SystemService` · `AiConfig`

- `SystemService` 维护 `FRONTEND_TO_DB_KEY` / `DB_TO_FRONTEND_KEY` 双向映射表
- 解决前端字段名（如 `model`）与数据库 key（如 `ai.chat-model`）不一致问题
- `updateConfig()` 保存到数据库后调用 `refreshAiConfig()` 刷新 `AiConfig` Bean
- **模型参数变更立即生效，无需重启服务**

#### 可动态配置项

| 配置项 | 数据库 key | 默认值 |
|--------|-----------|--------|
| API 地址 | `textModelApiUrl` | `https://api-inference.modelscope.cn/v1` |
| API Key | `textModelApiKey` | — |
| 模型 | `model` | `Qwen/Qwen3.5-35B-A3B` |
| 超时时间 | `timeout` | `30000` ms |
| 温度参数 | `temperature` | `0.3` |
| 最大 Token | `maxTokens` | `4096` |

### 7.6 评分流程

**核心类**: `ScoreService`

```
AI 自动评分 → 教师复核（逐项调整） → 修正记录 → 成绩发布
     │              │                    │            │
 score_result    teacher_score    score_correction  status=PUBLISHED
 (auto_score)    (覆盖写入)       (原始分→新分)
                                    ↳ 乐观锁 expectedUpdatedAt
```

- `saveTeacherScores()` 接收 4 个参数：`(submissionId, List<ScoreResult>, comment, expectedUpdatedAt)`
- `expectedUpdatedAt` 用于乐观锁，防止并发修改冲突

## 十一、部署指南

### 11.1 环境要求

| 依赖 | 最低版本 |
|------|----------|
| JDK | 17+ |
| Node.js | 18+ |
| MySQL | 8.0+ |
| Maven | 3.8+ |

### 11.2 构建步骤

#### 1. 初始化数据库

```sql
CREATE DATABASE bisai;
SOURCE backend/src/main/resources/schema.sql;
```

默认管理员：`admin` / `admin123`

#### 2. 启动后端

```bash
cd backend
mvn spring-boot:run    # 端口 8080
```

#### 3. 启动前端

```bash
cd frontend
npm install
npm run dev             # 端口 3000，自动代理 /api → localhost:8080
```

访问 `http://localhost:3000`

### 11.3 生产构建

```bash
# 后端
cd backend && mvn clean package -DskipTests
# → backend/target/backend-1.0.0.jar

# 前端
cd frontend && npm run build
# → frontend/dist/

# 部署
java -jar backend-1.0.0.jar
# 前端 dist/ 部署到 Nginx，反向代理 /api → 后端
```

### 11.4 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `DB_PASSWORD` | 数据库密码 | `123456` |
| `JWT_SECRET` | JWT 签名密钥 | 内置默认值 |
| `AI_API_KEY` | ModelScope API Key | 内置开发用 Key |

### 11.5 关键配置说明

- **热部署已禁用**：`devtools.restart.enabled: false`，修改 Java 代码需手动重启
- **MyBatis 日志**：使用 `Slf4jImpl`，避免定时任务刷屏
- **Jackson 全局格式化**：`JacksonConfig` 配置 `LocalDateTime` → `yyyy-MM-dd HH:mm:ss`
- **前端日期选择器**：必须 `value-format="YYYY-MM-DD HH:mm:ss"` 匹配后端格式
- **TypeScript 6.0.3**：`tsconfig.app.json` 保留 `"ignoreDeprecations": "6.0"` 避免 TS5101

## 十三、开发约定与注意事项

### 13.1 编码规范

- 前端中文界面，代码注释中文，变量名英文
- 后端 API 响应统一 `Result<T>` 包装
- 分页统一 `PageQuery` → `PageResult<T>`
- 状态标签集中管理在 `utils/status.ts`，**禁止在组件中硬编码**

### 13.2 关键陷阱

| 陷阱 | 解决方案 |
|------|----------|
| `updateById()` 不更新 null | 需清空字段时用 `UpdateWrapper.set("column", null)` |
| `spring.jackson.date-format` 对 `LocalDateTime` 无效 | 使用 `JacksonConfig` 处理 |
| 前端字段名 ≠ 数据库 key | `SystemService` 双向映射表 |
| 班级实体叫 `ClassEntity` 不是 `ClassInfo` | 引用时注意区分 |
| TS 6.0 `baseUrl` 废弃 | 保留 `"ignoreDeprecations": "6.0"` |
| `noUnusedLocals` / `noUnusedParameters` | 未使用导入会导致 `vue-tsc` 构建失败 |
| `ReportService` 文件路径 | 必须存绝对路径，`FileController` 会对相对路径拼接基础路径 |
