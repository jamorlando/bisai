# 实训成果智能核查与评价系统
# 演示 PPT 内容大纲

**时长建议：** 5-7 分钟  
**演示目标：** 展示项目价值、技术亮点、功能实现及国产化适配能力。

## Slide 1: 封面
*   **标题**：基于大模型技术的软件实训教学结果检查评价与报表系统
*   **副标题**：第十五届“中国软件杯”大学生软件设计大赛 - B1 赛题

*   **汇报人**：[你的名字]

## Slide 2: 痛点与背景
*   **痛点 1**：**教师批改累** - 批量查看代码、文档、截图，耗时耗力。
*   **痛点 2**：**标准主观强** - 不同教师对同一份作业评分可能不一致。
*   **痛点 3**：**反馈滞后** - 学生提交后需等待数天才能获知结果，失去了改进的最佳时机。
*   **我们的目标**：**自动化、智能化、标准化**。用 AI 赋能实训评价全流程。

## Slide 3: 系统架构设计
*   **架构图展示**：
    *   **前端**：Vue 3 + Element Plus (响应式、美观)。
    *   **后端**：Spring Boot 3 + Java 17 (企业级、高可用)。
    *   **AI 引擎**：Spring AI + ModelScope (Qwen 大模型 + RAG)。
*   **设计理念**：**业务系统本地部署 + AI 能力可插拔**。
    *   确保在断网或无本地算力环境下仍可降级运行。

## Slide 4: 核心功能演示 - 1/3
*   **多格式成果解析**
    *   支持 Word、PDF、Excel、ZIP、图片等多种格式。
    *   **亮点**：ZIP 安全沙箱（防路径穿越、限制大小/文件数）。
    *   *演示截图*：学生上传界面、文件列表。

## Slide 5: 核心功能演示 - 2/3
*   **智能核查与 RAG 评分**
    *   **核查**：自动对比任务要求，识别缺失章节和逻辑漏洞（高风险/中风险）。
    *   **评分**：结合 RAG 知识库（评分标准、优秀样例），生成指标得分和详细评语。
    *   *演示截图*：AI 核查结果页、评分理由详情。

## Slide 6: 核心功能演示 - 3/3
*   **教师复核与报表**
    *   **复核**：教师查看 AI 建议，手动调整分数，最终确认发布。
    *   **报表**：一键生成个人 Word/PDF 评价报告、班级 Excel 统计报表 (含 ECharts 图表)。
    *   *演示截图*：评分调整页、导出的 PDF 报告。

## Slide 7: 技术创新点
1.  **RAG 增强评分**：不仅是 AI “看”报告，更是结合知识库“懂”标准，评分有据可依。
2.  **异步任务调度**：基于数据库轮询 + 重试机制，保障大文件解析、批量评价不卡顿。
3.  **安全与防护**：Prompt 注入防御（三明治结构）、ZIP 沙箱、防爆破登录。
4.  **国产化无缝适配**：纯 Java 技术栈，0 Native 依赖，完美运行于 LoongArch + 麒麟 OS。

## Slide 8: 应用价值与推广
*   **对教师**：批改效率提升 **80%**，从“阅卷机器”变为“评价导师”。
*   **对学生**：即时反馈，精准定位薄弱环节，提升实训效果。
*   **对学校**：汇聚教学数据，生成“教学质量画像”，辅助教学改革。

## Slide 9: 部署与运行
*   **部署架构**：
    *   龙芯 3A5000 + 银河麒麟 V10。
    *   MySQL / OpenGauss 数据库。
    *   离线模式支持：AI 服务降级为纯人工评分，业务流不中断。

## Slide 10: 总结与展望
