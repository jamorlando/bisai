# 实训成果智能核查与评价系统
# 部署文档

**文档版本：** V1.0  
**适用环境：** 银河麒麟高级服务器操作系统 V10 (LoongArch) / Windows / Linux (AMD64)

---

## 一、环境准备

在部署前，请确保服务器满足以下最低配置要求：
*   **操作系统**：银河麒麟 V10 SP2 或更高版本。
*   **CPU**：Loongson 3A5000 或同级处理器 (4 核+)。
*   **内存**：16GB+。
*   **磁盘**：50GB+ 可用空间。

**依赖软件安装：**
1.  **JDK 17**：推荐安装支持 LoongArch 的 OpenJDK 17。
    ```bash
    # 麒麟源安装 (示例)
    sudo dnf install java-17-openjdk-loongarch64
    java -version
    ```
2.  **MySQL 8.0** (或 OpenGauss/Dameng)。
3.  **Nginx**：用于托管前端静态资源。

---

## 二、后端部署

### 2.1 数据库初始化
1.  登录数据库执行建表脚本：
    ```bash
    mysql -u root -p < backend/src/main/resources/schema.sql
    ```
2.  初始化默认管理员账号 (脚本中已包含)。

### 2.2 配置文件修改
编辑 `backend/src/main/resources/application.yml` (或打包后通过启动参数覆盖)：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/bisai?useUnicode=true&characterEncoding=utf8
    username: root
    password: ${DB_PASSWORD}
  ai:
    model: Qwen/Qwen3.5-35B-A3B
    base-url: https://api-inference.modelscope.cn/v1
    api-key: ${AI_API_KEY}
```

### 2.3 启动服务
```bash
# 方式 1: 直接运行 jar 包 (生产环境推荐)
java -jar -Xms1g -Xmx2g backend-1.0.0.jar

# 方式 2: 指定配置文件
java -jar backend-1.0.0.jar --spring.config.location=./application-prod.yml
```
*验证：访问 `http://localhost:8080`，返回 JSON 即成功。*

## 三、前端部署

### 3.1 构建前端 (在开发机执行)
```bash
cd frontend
npm install
npm run build
```
构建产物位于 `dist/` 目录。

### 3.2 Nginx 配置
将 `dist/` 下的文件复制到 Nginx 目录 (如 `/usr/share/nginx/html`)。
配置 `nginx.conf`：
```nginx
server {
    listen 80;
    server_name localhost;

    # 前端静态资源
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://localhost:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

## 四、国产化适配特别说明

### 4.1 LoongArch 注意事项
*   **字体问题**：导出 PDF/Word 时，麒麟服务器可能缺少中文字体。
    *   请安装：`sudo dnf install wqy-microhei-fonts`
    *   或在 `resources/fonts` 目录下放置 `.ttf` 字体文件。
*   **JNI 依赖**：本项目已避免使用依赖 x86 架构的 Native 库 (如 Tesseract)，全链路纯 Java/Node.js，确保无缝运行。

### 4.2 离线部署模式 (无外网)
若服务器无法访问互联网 (无法调用 ModelScope)：
1.  **AI 能力降级**：系统支持“纯人工评分模式”，不影响上传、查看和打分功能。
2.  **本地模型**：可部署 Ollama + Qwen2.5，修改 `ai.base-url` 指向本地端口 (如 `http://localhost:11434/v1`)。

## 五、常见问题排查

*   **启动报错 `Connection refused`**：检查 MySQL 是否启动及 `application.yml` 密码是否正确。
*   **AI 调用失败**：检查服务器能否访问外网，或 API Key 是否过期。
*   **中文乱码**：确保数据库字符集为 `utf8mb4`，且应用层设置了 UTF-8 编码。
